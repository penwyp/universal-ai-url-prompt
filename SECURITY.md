# Security Policy

## Reporting a Vulnerability

If you discover a security issue, do not open a public issue first.

- Preferred: create a private GitHub security advisory for this repository.
- If private advisory is not available, open a minimal issue without exploit details and request a private contact channel.

## Response Targets

- Initial acknowledgment: within 3 business days.
- Triage decision: within 7 business days.
- Fix release target: depends on severity and scope.

## Disclosure

Please allow maintainers time to investigate and release a fix before public disclosure.
